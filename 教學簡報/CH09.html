這一章要做出什麼

即時模式：不用等你說完

逐句模式要「說完按停」；即時模式邊說就邊翻、直接吐語音，延遲很低。我們接 Gemini Live 這個「原生語音」的即時模型來做。

🎁 本章小成品

一個「即時口譯」切換：按下去持續聽、持續翻、持續說，中英夾雜也能自動雙向翻。

😤 先講清楚：這是全課最難的一章。它牽涉到「即時連線」「原始聲音」「多執行緒」。別慌——我們會把每一小塊拆開，一步一步驗證。你只要照著貼、照著看，就能通關。

逐句 vs 即時：差在哪

兩種模式各有用途，不是誰取代誰
逐句（CH07-08）即時（本章）

白話觀念（一）：WebSocket 與串流

先把陌生名詞講成人話
📞

WebSocket

一種「講電話」式的連線。一般網頁是「你問一句、它答一句」就掛斷；WebSocket 是一直連著，雙方隨時能講話。即時口譯要一直傳聲音，所以用它。

🌊

串流（streaming）

不是「等整段錄好才送」，而是像水流一樣邊產生邊送。你講一點就送一點，AI 翻一點就回一點，所以感覺很即時。

💡 這章我們用 flask-socketio 這個套件來做 WebSocket，前端則用 socket.io 來連。名字很像，是一組的。

白話觀念（二）：PCM 與 16kHz

聲音在電腦裡長什麼樣
🎚️

PCM

「沒壓縮的原始聲音」。MP3 是壓過的（檔案小、但要解壓）；PCM 是最原始的一串數字，直接就是聲音的波形。即時傳輸不想花時間壓縮解壓，所以用 PCM。

🔢

16kHz 取樣

「每秒鐘量 16000 次聲音大小」。量越多越清楚、檔越大。人聲用 16kHz 就夠清楚了，這也是 Gemini Live 要求送進去的規格。

🔎 小記：送進去用 16kHz PCM，回來的語音是 24kHz PCM（品質更好一點）。這兩個數字後面會用到，先有個印象就好。

# app.py — GeminiSession：一位使用者一份即時對話
class GeminiSession:
    def __init__(self, sid, instructions, api_key=None):
        self.sid = sid                          # 是誰（哪個瀏覽器）
        self.instructions = instructions        # 翻譯指令
        self.audio_in_queue = queue.Queue()      # 你送來的音訊排隊處
        self.stop_event = threading.Event()      # 按停止用的旗子
        self.thread = None
        self.client = genai.Client(api_key=(api_key or API_KEY),
                                   http_options={'api_version': 'v1alpha'})

    def start(self):
        self.thread = threading.Thread(target=self.run_loop)
        self.thread.start()                      # 開背景執行緒

    def stop(self):
        self.stop_event.set()                    # 舉旗：該收工了
        if self.thread: self.thread.join(timeout=2)

    def add_audio(self, audio_data):
        self.audio_in_queue.put(audio_data)      # 音訊丟進佇列

    def run_loop(self):
        asyncio.run(self.async_process())        # 在執行緒裡跑 async

步驟三：AI 會給你這段（中半：真正連 Gemini）

async_process：接上專線
    async def async_process(self):
        config = {
            "response_modalities": ["AUDIO"],        # 原生語音只吐 AUDIO
            "system_instruction": self.instructions,
            "output_audio_transcription": {},        # 順便要字幕
        }
        try:
            async with self.client.aio.live.connect(
                    model=LIVE_MODEL, config=config) as session:
                logger.info(f"Session {self.sid} connected to Gemini Live.")
                sender_task = asyncio.create_task(self.sender(session))
                receiver_task = asyncio.create_task(self.receiver(session))
                while not self.stop_event.is_set():   # 沒喊停就一直跑
                    await asyncio.sleep(0.1)
                sender_task.cancel(); receiver_task.cancel()
        except Exception as e:
            logger.error(f"Gemini connection error: {e}")
            socketio.emit('error', {'msg': str(e)}, to=self.sid)
⚠️ 進階，先跳過不影響主線：實際專案裡這個 while 迴圈還會「幫掛掉的 sender/receiver 復活」（自動重啟）。那是穩定性優化，看不懂沒關係，照抄能跑就好。

步驟三：AI 會給你這段（下半：送與收）

sender 往上送、receiver 往下傳
    async def sender(self, session):        # 把你的 PCM 送給 Gemini
        from google.genai.types import Blob
        while True:
            if not self.audio_in_queue.empty():
                chunk = self.audio_in_queue.get()
                blob = Blob(data=chunk, mime_type="audio/pcm")
                await session.send_realtime_input(audio=blob)
            else:
                await asyncio.sleep(0.01)

    async def receiver(self, session):      # 收 Gemini 回來的東西
        async for response in session.receive():
            sc = response.server_content
            if sc is None: continue
            ot = getattr(sc, 'output_transcription', None)
            if ot and getattr(ot, 'text', None):
                socketio.emit('text_response', {'text': ot.text}, to=self.sid)  # 字幕
            if sc.model_turn:
                for part in sc.model_turn.parts:
                    inline = getattr(part, 'inline_data', None)
                    if inline and inline.data:
                        socketio.emit('audio_response', inline.data, to=self.sid) # 24kHz 語音
            if sc.turn_complete:
                socketio.emit('turn_complete', to=self.sid)   # 一輪結束

步驟三：逐段看懂

整個 GeminiSession 的分工
角色在做什麼
audio_in_queue（佇列）你送來的音訊先排隊，sender 再一份份拿去送。像「收件匣」
start() 開執行緒即時連線要一直跑，不能卡住主程式，所以丟到背景執行緒
live.connect(...)真正撥通到 Gemini Live 的即時專線
sender不停檢查佇列，有音訊就 send_realtime_input 送上去
receiver不停收 Gemini 回來的：語音→audio_response、字幕→text_response
stop_event一面「該收工了」的旗子，按停止時舉起來，迴圈就結束
🧩 一句話總結：一位使用者 = 一個 GeminiSession = 一條執行緒 = 一通到 Gemini 的電話。

@socketio.on('start_session')
def handle_start(data):
    sid = request.sid
    langA = data.get('langA', 'Chinese'); langB = data.get('langB', 'English')
    user_key = (data.get('gemini_key') or '').strip() or None
    instruction = (
        f"You are a real-time voice translator. Your ONLY job is to translate speech.\n"
        f"- When you hear {langA}, translate it to {langB} and reply in {langB}.\n"
        f"- When you hear {langB}, translate it to {langA} and reply in {langA}.\n"
        f"Rules: Output ONLY the translation. No greeting or explanation. Speak naturally.")
    if sid in active_sessions: active_sessions[sid].stop()
    session = GeminiSession(sid, instruction, api_key=user_key)
    active_sessions[sid] = session; session.start()
    emit('status', {'msg': 'Session Started'})

步驟四：另外三個事件

送音訊 / 停止 / 斷線
# app.py — 續前
@socketio.on('audio_in')                # 前端一直送來的音訊
def handle_audio(data):
    sid = request.sid
    if sid in active_sessions:
        active_sessions[sid].add_audio(data)   # 丟進那位使用者的佇列

@socketio.on('stop_session')            # 按停止
def handle_stop():
    sid = request.sid
    if sid in active_sessions:
        active_sessions[sid].stop()
        del active_sessions[sid]

@socketio.on('disconnect')              # 關網頁 / 斷線也要收工
def handle_disconnect():
    sid = request.sid
    if sid in active_sessions:
        active_sessions[sid].stop(); del active_sessions[sid]
    logger.info(f"Client disconnected: {sid}")

步驟五：AI 會給你這段（startLive）

貼進 static/js/app.js
let liveOn = false, audioCtx, liveProcessor, liveInput, liveStream;

async function startLive(){
  liveStream = await navigator.mediaDevices.getUserMedia({ audio:true });
  audioCtx = audioCtx || new (window.AudioContext||window.webkitAudioContext)();
  liveInput = audioCtx.createMediaStreamSource(liveStream);
  liveProcessor = audioCtx.createScriptProcessor(4096, 1, 1);
  liveInput.connect(liveProcessor); liveProcessor.connect(audioCtx.destination);
  const targetRate = 16000;                    // Gemini 要 16kHz
  liveProcessor.onaudioprocess = (e)=>{
    if(!liveOn) return;
    const input = e.inputBuffer.getChannelData(0);
    const step = audioCtx.sampleRate / targetRate;   // 降採樣的步進
    const len = Math.floor(input.length / step);
    const pcm = new Int16Array(len);
    for(let i=0;i<len;i++){
      let s = Math.max(-1, Math.min(1, input[Math.floor(i*step)]||0));
      pcm[i] = s < 0 ? s*0x8000 : s*0x7FFF;    // -1~1 轉 16-bit 整數
    }
    socket.emit('audio_in', pcm.buffer);        // 一份份送出
  };
  socket.emit('start_session', { langA:'Chinese', langB:'English' });
  liveOn = true; setResult(sResult, '即時聆聽中…', true);
}

📂 一樣在 static/js/app.js 的 // ===== 5. 即時語音 ===== 區，接在 startLive() 後面。💡 playLiveAudio() 就是 CH08 朗讀 TTS 已經寫好的同一個播放函式，這裡直接沿用、不用重寫。

// app.js — 綁定 Gemini 回來的三種訊號（同一區）
let livePending = '';
socket.on('text_response', (d)=>{ if(d.text){ livePending += d.text; setResult(sResult, livePending); } });  // 字幕逐字出現
socket.on('audio_response', (data)=> playLiveAudio(data));   // 播放翻譯語音
socket.on('turn_complete', ()=>{ livePending = ''; });        // 一輪講完，字幕清空

// 播放 Gemini 原生語音（24kHz PCM 16-bit）
let playCtx = null, nextTime = 0;
function playLiveAudio(data){
  if(!playCtx) playCtx = new (window.AudioContext||window.webkitAudioContext)({ sampleRate:24000 });
  if(playCtx.state==='suspended') playCtx.resume();   // 手機須先解鎖才有聲
  const int16 = new Int16Array(data);
  const f32 = new Float32Array(int16.length);
  for(let i=0;i<int16.length;i++) f32[i] = int16[i] / 32768;  // 整數轉回 -1~1
  const buf = playCtx.createBuffer(1, f32.length, 24000);
  buf.getChannelData(0).set(f32);
  const src = playCtx.createBufferSource();
  src.buffer = buf; src.connect(playCtx.destination);
  const now = playCtx.currentTime; if(nextTime < now) nextTime = now;
  src.start(nextTime); nextTime += buf.duration;      // 一段接一段，不會疊在一起
}

步驟五：逐段看懂播放邏輯

為什麼要 nextTime 這一招
這幾行在做什麼
new Int16Array(data)把後端傳來的一串 bytes 讀成「16-bit 整數」的聲音樣本
int16[i] / 32768整數轉回 -1~1 的小數，這是 Web Audio 要的格式
createBuffer(1, len, 24000)建一段 24kHz、單聲道的音訊緩衝（回來的語音是 24kHz）
nextTime 排程語音是一小段一小段回來的。用 nextTime 把每段接龍排好，才不會全部擠在同一刻播、聽起來像雜訊
🔊 這就是「串流播放」的精髓：邊收邊排、排好就放，聽感上就像連續一句話。

// app.js — 一顆按鈕切換即時開/關（同一區）
async function toggleLive(){ liveOn ? stopLive() : startLive(); }

function stopLive(){
  liveOn = false;
  if(liveProcessor){ liveProcessor.disconnect(); liveProcessor = null; }
  if(liveInput){ liveInput.disconnect(); liveInput = null; }
  if(liveStream){ liveStream.getTracks().forEach(t=>t.stop()); liveStream = null; }  // 關麥克風燈
  if(socket) socket.emit('stop_session');
}

// index.html 放一顆按鈕：<button onclick="toggleLive()">🎙️ 即時</button>
🧹 stopLive() 很重要：不關麥克風，瀏覽器那顆紅點會一直亮、也會一直耗流量。務必把三個東西（processor／input／stream）都收乾淨。

對症下藥
① 麥克風開不了 / getUserMedia 失敗：正式網址一定要 HTTPS；本機測用 http://127.0.0.1:5001 是被允許的。瀏覽器要按「允許」麥克風。

③ 延遲很大 / 聲音斷斷續續：多半是網路。也確認 PCM 有正確降到 16kHz（步驟五的 step 算式）。
④ 翻譯行為怪 / 亂翻：可暫時把模型換成通用即時模型 gemini-3.1-flash-live-preview 比較差異（通用版，非專為翻譯優化）。
⑤ 回 429 / 額度：即時模式較耗量，Gemini 免費額度容易到上限，等一下或換金鑰。
