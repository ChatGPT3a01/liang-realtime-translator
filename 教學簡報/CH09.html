這一章要做出什麼

即時模式：不用等你說完

逐句模式要「說完按停」；即時模式邊說就邊翻、直接吐語音，延遲很低。我們接 Gemini Live 原生語音來做。

🎁 本章小成品

一個「即時口譯」切換：按下去持續聽、持續翻、持續說。

逐句 vs 即時

兩種模式各有用途
逐句（CH07-08）即時（本章）
時機說完按停才翻邊說邊翻
延遲較高很低
技術瀏覽器 STT + 翻譯 + TTSGemini Live 原生語音串流
連線一般 HTTPWebSocket 即時雙向

即時模式的資料流

先懂原理再動手
你的麥克風
   ↓ 擷取音訊、轉成 16kHz PCM
瀏覽器 → (WebSocket) → 你的 Flask 伺服器
   ↓ 轉送
Gemini Live（即時翻譯模型）
   ↓ 回傳翻譯後的語音 + 字幕
Flask → (WebSocket) → 瀏覽器
   ↓ 播放 24kHz 語音、顯示字幕

前端：擷取麥克風 PCM

轉成 16kHz 送出去
// app.js（即時模式）
const socket = io();
async function startLive(){
  const stream = await navigator.mediaDevices.getUserMedia({audio:true});
  const ctx = new AudioContext();
  const src = ctx.createMediaStreamSource(stream);
  const proc = ctx.createScriptProcessor(4096,1,1);
  src.connect(proc); proc.connect(ctx.destination);
  proc.onaudioprocess = (e)=>{
    const input = e.inputBuffer.getChannelData(0);
    const pcm = downsampleTo16k(input, ctx.sampleRate);  // 轉 16kHz Int16
    socket.emit('audio_in', pcm.buffer);
  };
  socket.emit('start_session', {langA:'Chinese', langB:'English'});
}

常見問題

即時模式的雷

連不上 / 沒聲音？

確認 socketio.run 啟動、前端有載 socket.io、麥克風權限允許。

延遲很大？

網路品質影響大；PCM 取樣要正確降到 16kHz。

翻譯行為怪？

若即時翻譯品質或行為異常，可回退到通用即時模型 gemini-3.1-flash-live-preview（通用版，非專為翻譯優化）比較差異。

額度到了？

即時模式較耗用量，注意 Gemini 支出上限。
