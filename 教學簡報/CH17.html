設定面板（localStorage）

存在使用者瀏覽器
// app.js — 讀寫設定
const DEFAULT_CFG = { provider:'gemini', geminikey:'', tavilykey:'', owmkey:'' };
function loadCfg(){ return {...DEFAULT_CFG, ...JSON.parse(localStorage.getItem('cfg')||'{}')}; }
function saveCfg(c){ localStorage.setItem('cfg', JSON.stringify(c)); }
let cfg = loadCfg();
// 請求時帶上（Gemini 空的話後端用伺服器內建）
function providerBody(){ return { provider:cfg.provider, api_key:cfg.geminikey||'' }; }

後端各端點改成「有帶金鑰就用、沒帶就用伺服器內建」，彈性最大。

密碼閘門怎麼做？

用 SHA-256 雜湊，不存明文

把「正確密碼」的 SHA-256 雜湊寫在程式裡。使用者輸入密碼時，前端算它的雜湊，跟存好的比對，對了才放行。

⚠️ 這是「閱讀阻擋」等級，不是高強度保護（前端的東西使用者都看得到）。真正機密仍要靠後端與環境變數。

密碼閘門程式

比對雜湊
// 只存雜湊，不存明文密碼
const HASHES = ['3ee7f6...要換成你的密碼雜湊'];
async function sha256(str){
  const buf = await crypto.subtle.digest('SHA-256', new TextEncoder().encode(str));
  return [...new Uint8Array(buf)].map(b=>b.toString(16).padStart(2,'0')).join('');
}
async function check(pw){
  const h = await sha256(pw);
  if(HASHES.includes(h)){ sessionStorage.setItem('authed','1'); hideGate(); }
  else alert('密碼錯誤');
}

🎁 SHA-256 密碼雜湊生成器

輸入密碼，算出雜湊貼進程式
⚙️ 產生密碼雜湊
輸入密碼後按「算出 SHA-256」，這裡會出現雜湊值…
把雜湊值放進程式的 HASHES 陣列。程式碼裡看不到明文密碼，較安全。

實測

設定 + 密碼都試一遍

用生成器算出你的密碼雜湊、放進 HASHES；重整頁面應先跳密碼閘門，輸入正確密碼才進站。到設定面板填一把金鑰，確認翻譯仍正常。
