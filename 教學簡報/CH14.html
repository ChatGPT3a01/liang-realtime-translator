先看全貌：查一個地名，中間發生什麼事

看懂這條路，後面每一步就知道自己在做哪一段
  1. 使用者按「🌤️ 天氣」，打上地名（例如「那霸」）按「查詢」
  2. 前端把地名送到後端的 /api/weather
  3. 後端先地理編碼：把地名換成經緯度（天氣 API 只吃座標）
  4. 後端拿座標去問 Open-Meteo，拿到溫度、天氣代碼、降雨機率
  5. 後端把天氣代碼翻成中文、算出「帶傘建議」，一起回傳
  6. 前端把地名、溫度、天氣、建議顯示在畫面上
👉 這一章就是照著這 6 步，一段一段做出來。

# app.py — 內建字典 → Nominatim → Open-Meteo；查無回 None
def _geocode(place):
    key = place.strip()
    if key in COMMON_PLACES:                     # ① 先查白名單
        return COMMON_PLACES[key]
    try:                                         # ② Nominatim（中文佳，須帶 User-Agent）
        r = requests.get("https://nominatim.openstreetmap.org/search",
            params={"q": place, "format": "json", "limit": 1, "accept-language": "zh-TW"},
            headers={"User-Agent": "liang-translator/1.0 (travel weather)"}, timeout=10)
        d = r.json()
        if d:
            item = d[0]
            disp = item.get("display_name", place).split(",")
            name = disp[0].strip() + (("，" + disp[-1].strip()) if len(disp) > 1 else "")
            return float(item["lat"]), float(item["lon"]), name
    except Exception as e:
        logger.warning(f"nominatim failed: {e}")
    try:                                         # ③ Open-Meteo geocoding（最後備援）
        g = requests.get("https://geocoding-api.open-meteo.com/v1/search",
            params={"name": place, "count": 1, "language": "zh"}, timeout=8).json()
        results = g.get("results") or []
        if results:
            loc = results[0]
            name = loc.get("name", place) + (("，" + loc["country"]) if loc.get("country") else "")
            return loc["latitude"], loc["longitude"], name
    except Exception as e:
        logger.warning(f"open-meteo geocode failed: {e}")
    return None

步驟一：逐段看懂

三層備援，一層比一層兜底
這幾行在做什麼
if key in COMMON_PLACES先查內建白名單。命中就直接回座標，最準、最快、零網路
requests.get("…/nominatim…")白名單沒有 → 問 OpenStreetMap 的 Nominatim，它中文地名支援好
headers={"User-Agent": …}Nominatim 規定一定要帶「你是誰」的標頭，不帶會被擋
display_name.split(",")把「那霸市, 沖繩縣, 日本」切一切，組出乾淨的顯示名
geocoding-api.open-meteo…前兩個都失敗 → Open-Meteo 官方地理編碼當最後救援

🧱 這叫「多層備援（fallback）」：一條路不通就換下一條，讓查詢盡量成功。

步驟一 · 一定要教的坑：中文地名查不到

為什麼要做三層備援

如果只用 Open-Meteo 一家的地理編碼，你會踩到一個很常見的雷：它對中文地名很不準——「那霸」「墾丁」「峴港」常常查不到，對台灣使用者是大問題。

✅ 解法就是這三層：① 內建字典保證熱門地點精準 → ② Nominatim 中文支援最好 → ③ Open-Meteo 兜底。所以「那霸」一定查得到。
🔧 卡住了？ 若某個地名還是查不到 → 改用英文地名或加上國家，例如把「峴港」改成 Da Nang、把「首爾」改成 Seoul, South Korea。這是最實用的除錯招數，下面除錯集會再示範。

# app.py — 依天氣代碼、溫度、降雨機率給建議
def _weather_advice(code, temp, pop):
    tips = []
    rainy = code in (51, 53, 55, 56, 57, 61, 63, 65, 66, 67,
                     80, 81, 82, 95, 96, 99) or (pop is not None and pop >= 50)
    if rainy:
        tips.append("☂️ 建議帶傘")           # 下雨代碼 或 降雨機率≥50%
    if temp is not None:
        if temp >= 30:
            tips.append("🧴 高溫，注意防曬補水")
        elif temp <= 12:
            tips.append("🧥 偏冷，記得保暖")
    if not tips:
        tips.append("👍 天氣舒適，玩得開心")
    return "，".join(tips)                    # 多條建議用「，」串起來

# app.py
@app.route('/api/weather', methods=['POST'])
def api_weather():
    data = request.get_json(force=True, silent=True) or {}
    place = (data.get('place') or '').strip()
    lat, lon = data.get('lat'), data.get('lon')
    name = place or "目前位置"
    try:
        if place:                              # 有地名先地理編碼
            geo = _geocode(place)
            if not geo:
                return jsonify({"ok": False, "error": f"找不到地點「{place}」"}), 400
            lat, lon, name = geo
        if lat is None or lon is None:
            return jsonify({"ok": False, "error": "缺少地點或座標"}), 400

        w = requests.get("https://api.open-meteo.com/v1/forecast", params={
            "latitude": lat, "longitude": lon,
            "current": "temperature_2m,relative_humidity_2m,apparent_temperature,weather_code,precipitation",
            "daily": "temperature_2m_max,temperature_2m_min,precipitation_probability_max",
            "timezone": "auto", "forecast_days": 1,
        }, timeout=8).json()
        cur, daily = w.get("current") or {}, w.get("daily") or {}
        code = cur.get("weather_code")
        temp = cur.get("temperature_2m")
        pop = (daily.get("precipitation_probability_max") or [None])[0]
        return jsonify({
            "ok": True, "place": name, "temp": temp,
            "feels": cur.get("apparent_temperature"),
            "humidity": cur.get("relative_humidity_2m"),
            "desc": WMO_CODES.get(code, "—"), "pop": pop,
            "hi": (daily.get("temperature_2m_max") or [None])[0],
            "lo": (daily.get("temperature_2m_min") or [None])[0],
            "advice": _weather_advice(code, temp, pop),
        })
    except Exception as e:
        logger.error(f"weather error: {e}")
        return jsonify({"ok": False, "error": f"天氣查詢失敗：{type(e).__name__}"}), 400

// static/js/app.js — 送 payload 給後端，畫出結果
async function fetchWeather(payload) {
  $('wx_status').textContent = '查詢中…';
  try {
    const res = await fetch('/api/weather', {
      method: 'POST', headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ ...payload, owm_key: cfg.owmkey || '' }),
    });
    const d = await res.json();
    if (!d.ok) throw new Error(d.error || '查詢失敗');
    $('wx_result').innerHTML =
      `<div class="wx-place">${d.place}</div>` +
      `<div class="wx-temp">${Math.round(d.temp)}°C　${d.desc}</div>` +
      `<div class="wx-advice">${d.advice}</div>`;
    $('wx_result').classList.remove('hidden');
    $('wx_status').textContent = '';
  } catch (e) { $('wx_status').textContent = '❌ ' + e.message; }
}
// 按「查詢」→ 用地名查
$('wx_go').addEventListener('click', () => {
  const p = $('wx_place').value.trim();
  if (!p) { $('wx_status').textContent = '請輸入地點'; return; }
  fetchWeather({ place: p });
});
// 按「用目前位置」→ 瀏覽器定位，傳經緯度
$('wx_geo').addEventListener('click', () => {
  navigator.geolocation.getCurrentPosition(
    (pos) => fetchWeather({ lat: pos.coords.latitude, lon: pos.coords.longitude }),
    () => { $('wx_status').textContent = '無法取得位置（請允許定位權限）'; });
});

🔧 卡住了？本章常見的坑

對症下藥
① 中文地名查不到（最常見）：某個地名回「找不到地點」→ 先確認有沒有在 COMMON_PLACES；沒有就改用英文地名或加國家，例如「峴港」→ Da Nang、「清邁」→ Chiang Mai, Thailand。
② Nominatim 一直失敗：多半是沒帶 User-Agent，或短時間查太多次被限流。確認 headers={"User-Agent": …} 有加、查詢別太密集。
③ 「用目前位置」沒反應：定位要在 https 或 localhost 才能用，且瀏覽器要允許定位權限。被拒就會顯示「無法取得位置」。
④ 溫度顯示 NaN／—：多半是網路擋掉了 Open-Meteo，或座標怪怪的。先在瀏覽器直接開 api.open-meteo.com 網址測試連得到嗎。

除錯實戰：中文查不到就換英文

一個很實用的排錯步驟

假設你查「峴港」回「找不到地點」。別急著改程式，先換一種寫法再查一次：

查不到（中文）改這樣查（英文／加國家）
峴港Da Nang
清邁Chiang Mai, Thailand
宿霧Cebu, Philippines
✅ 若換英文就查到了 → 代表是地理編碼「認不得中文」，不是你程式壞了。想一勞永逸，就把這個城市加進 COMMON_PLACES（用最後的生成器）。

動手練習

改一點點，感受差異
  • 把 _weather_advice 裡的高溫門檻 temp >= 30 改成 28，再查曼谷，看提醒是否更早跳出。
  • 把降雨帶傘門檻 pop >= 50 改成 30，體會「靈敏 vs 誤報」的取捨。
  • 故意查一個奇怪地名（如「阿亮村」），確認會優雅地回「找不到地點」而不是整個當掉。
  • 用最後的生成器，把你下次要去的城市加進 COMMON_PLACES。
