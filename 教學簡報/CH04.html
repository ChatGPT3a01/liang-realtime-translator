什麼是 Flask？

用 Python 做網站的輕量框架

Flask 是一個 Python 的網站框架。它幫你處理「有人連到某個網址時，要回什麼」這件事。

它很輕、很好學，幾行程式就能跑起一個網站，非常適合初學者與這種專案。

🧩 你只要定義「網址 → 要回傳什麼」，Flask 負責其餘的網路細節。

app.py（骨架：先分好 10 區）

整支檔案先畫好格子，之後每章填一格
# app.py —— 整支檔案先分成 10 區，之後每一章填其中一區
import os
from flask import Flask, jsonify, render_template
from dotenv import load_dotenv

# ===== 1. 設定與金鑰 =====   （CH04，這章先放一點）
load_dotenv()                      # 讀取 .env 裡的設定
app = Flask(__name__)

# ===== 2. 翻譯 API =====          （CH06 會填 /api/translate）
# ===== 3. 朗讀 TTS API =====      （CH08 會填 /api/tts）
# ===== 4. 即時語音 SocketIO =====（CH09 會填）
# ===== 5. 拍照 / 檔案 API =====   （CH11–12 會填）
# ===== 6. 匯率 API =====          （CH13 會填 /api/currency）
# ===== 7. 天氣 API =====          （CH14 會填 /api/weather）
# ===== 8. 旅遊助手 API =====      （CH15 會填）
# ===== 9. PWA 與路由 =====        （CH16 會填 / 與靜態檔）

# ===== 10. 啟動 =====
@app.route('/')
def index():
    return 'Hello 亮言！我的翻譯 App 開工了 🚀'

@app.route('/api/health')
def health():
    return jsonify({'ok': True})

if __name__ == '__main__':
    port = int(os.getenv('PORT', '5001'))
    app.run(host='0.0.0.0', port=port, debug=True)

逐段看懂 app.py

不必背，抓重點就好
  • load_dotenv()：把 .env 裡的金鑰讀進來備用。
  • app = Flask(__name__)：建立網站應用程式。
  • @app.route('/')：有人連到首頁時，執行下面的函式、回傳文字。
  • /api/health：回傳 JSON，之後用來檢查伺服器活著沒。
  • app.run(...)：啟動伺服器，監聽 5001 埠。

常見問題

跑不起來先看這

找不到 flask？

套件沒裝或裝錯環境。重跑 pip install -r requirements.txt。

Port 5001 被占用？

改用別的埠：把 app.py 的 5001 改成 5002，或關掉占用的程式。

改了程式沒變化？

存檔（Ctrl/Cmd+S），debug 模式會自動重載；必要時 Ctrl+C 停掉再 python app.py。

要停止伺服器？

在終端機按 Ctrl + C。

認識「路由」

網址 → 對應的函式

每個 @app.route('網址') 就是一條「路由」：有人連到那個網址，就執行底下的函式、回傳結果。

我們的 App 之後會有很多路由：/api/translate、/api/weather、/api/currency…每個負責一個功能。

🛣️ 你可以把後端想成一棟房子，每條路由是一個窗口，各自處理不同的服務。

把金鑰讀進來（預備）

.env + load_dotenv 的組合

還記得 CH02 生成的 .env 嗎？把它放到專案根目錄。load_dotenv() 會讀它，之後程式就能這樣拿金鑰：

# 之後在需要的地方這樣取用
API_KEY = os.getenv('GEMINI_API_KEY')

現在骨架還用不到金鑰，但管線已經接好，CH06 串翻譯時就直接用。

順手建 .gitignore

先講，之後上傳 GitHub 才不會外洩金鑰

建一個檔案叫 .gitignore，列出「不要上傳」的東西。最重要的是 .env：

.env
__pycache__/
*.pyc
🔐 有了這行，你的金鑰檔就不會被傳上 GitHub。安全習慣，從第一天養成。
