白話觀念：什麼是「免金鑰 API」

這是本課的省錢重點之一

前面接 Gemini，我們得先去申請一把「金鑰」（API Key）當通行證。但有些公開服務不用金鑰、不用註冊、不用付錢就能直接查資料——這就是「免金鑰 API」。

匯率就是這種資料：全世界都一樣，沒有隱私問題，所以有好心的服務直接免費開放查詢。

💰 為什麼強調免金鑰？因為它代表：不用申請帳號、不怕忘記續費、不會不小心把金鑰外洩。功能照樣做出來，成本是零。這是我們整堂課「用最省的方式做出實用功能」的核心精神。

白話觀念：什麼是「主來源 + 備援」

一家掛了，另一家頂上

免費服務偶爾會「臨時罷工」（維修、塞車、暫時連不上）。如果我們只靠一家，它一掛，我們的換算就壞了。

所以我們準備兩家：先問主來源，問到就用；問不到才去問備援。這種「Plan B」設計，工程上叫容錯（fallback）。

🪂

容錯 = 備用降落傘

主傘（主來源）沒開，還有備傘（備援）。使用者根本不會發現剛剛出過事。

我們用的兩家免金鑰匯率來源

主用 ER-API，備援 Frankfurter
💵

ER-API（主來源）

open.er-api.com，免金鑰、幣別多，含台幣 TWD，還會回「最後更新時間」。

🏦

Frankfurter（備援）

frankfurter.app，用歐洲央行資料，免金鑰。主來源失敗時頂上。

⚠️ 為什麼主來源是 ER-API 而不是 Frankfurter？因為 Frankfurter 是歐洲央行資料，沒有台幣 TWD（也缺部分亞幣）。我們是台灣人，最常查台幣，所以把「有 TWD」的 ER-API 放第一順位。

# app.py — 匯率換算：免金鑰，主用 er-api，備援 frankfurter
# 檔案上方要有 import requests（前面章節已加過）
@app.route('/api/currency', methods=['POST'])
def api_currency():
    data = request.get_json(force=True, silent=True) or {}
    base = (data.get('base') or 'USD').upper()
    target = (data.get('target') or 'TWD').upper()
    try:
        amount = float(data.get('amount', 1) or 1)
    except (TypeError, ValueError):
        amount = 1.0

    if base == target:                                    # 同幣別免查，直接回
        return jsonify({"ok": True, "base": base, "target": target,
                        "amount": amount, "rate": 1.0, "result": amount,
                        "date": "", "source": "same"})

    # 主：open.er-api.com（免金鑰，幣別多，含 TWD）
    try:
        r = requests.get(f"https://open.er-api.com/v6/latest/{base}", timeout=8)
        r.raise_for_status()
        d = r.json()
        rate = (d.get("rates") or {}).get(target)
        if rate:
            return jsonify({"ok": True, "base": base, "target": target,
                "amount": amount, "rate": rate, "result": amount * rate,
                "date": d.get("time_last_update_utc", ""), "source": "er-api"})
    except Exception as e:
        logger.warning(f"currency er-api failed: {e}")     # 主來源掛了，往下退

    # 備援：frankfurter.app（歐洲央行，無 TWD 等部分亞幣）
    try:
        r = requests.get("https://api.frankfurter.app/latest",
                         params={"from": base, "to": target}, timeout=8)
        r.raise_for_status()
        d = r.json()
        rate = (d.get("rates") or {}).get(target)
        if rate:
            return jsonify({"ok": True, "base": base, "target": target,
                "amount": amount, "rate": rate, "result": amount * rate,
                "date": d.get("date", ""), "source": "frankfurter"})
    except Exception as e:
        logger.warning(f"currency frankfurter failed: {e}")

    return jsonify({"ok": False,
        "error": f"查不到 {base}→{target} 匯率（請確認幣別代碼）"}), 400

收資料 + 同幣別捷徑
這幾行在做什麼
base = (data.get('base') or 'USD').upper()拿來源幣別，沒給就預設美元；.upper() 轉大寫（jpy→JPY，API 只認大寫）
target = ... or 'TWD'拿目標幣別，沒給就預設台幣

步驟一：逐段看懂 (2)

主來源 ER-API 怎麼查

requests.get(f"https://open.er-api.com/v6/latest/{base}", timeout=8)：向主來源要「以 base 為基準的所有匯率表」。timeout=8 是「最多等 8 秒」，避免它卡住讓使用者一直等。

r.raise_for_status()：如果對方回錯誤（例如 404、500），這行會主動丟出例外，好讓下面的 except 接住、往備援退。

rate = (d.get("rates") or {}).get(target)：回來的資料是一大張匯率表，我們從裡面撈出「目標幣別」那一格。(... or {}) 是保險：萬一沒有 rates 這欄，也不會整個爆掉。

if rate:：真的撈到匯率才回結果（含 result = amount * rate 算好的金額、更新時間）。撈不到就不回，讓程式繼續往下走到備援。

步驟一：逐段看懂 (3) · 本章精華

「主來源失敗 → 自動退備援」是怎麼發生的

關鍵在那句 except Exception as e: logger.warning(...)：主來源只要出任何狀況（連不上、逾時、回錯誤），程式不會停，只記一筆警告，然後自然地往下走到 Frankfurter 那段。

備援 Frankfurter 的查法幾乎一樣，只是它的網址帶 from、to 參數。若它也查到就回結果；若兩家都查不到，最後才回 ok:false 加一句人話錯誤。

✅ 這就是「容錯」的完整長相：try 主來源 → 失敗吞掉 → try 備援 → 都失敗才認輸。使用者多數時候根本不會感覺到主來源曾經掛過。
🔧 卡住了？　若你把主來源那段的 try/except 拿掉，主來源一掛，整個 /api/currency 會直接回 500 錯誤、永遠退不到備援——這就是為什麼每一家都要各自包一層 try。

let curTimer = null;
async function doConvert() {
    const amount = parseFloat($('cur_amount').value);
    const from = $('cur_from').value, to = $('cur_to').value;
    if (!isFinite(amount)) { $('cur_result').textContent = '請輸入金額'; return; }
    $('cur_result').textContent = '換算中…';
    try {
        const res = await fetch('/api/currency', {
            method: 'POST', headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ base: from, target: to, amount }),
        });
        const d = await res.json();
        if (!d.ok) throw new Error(d.error || '查詢失敗');
        $('cur_result').textContent = `${amount} ${from} = ${d.result.toFixed(2)} ${to}`;
        const when = (d.date || '').replace(' (UTC)', '').slice(0, 16);
        $('cur_rate').textContent = `1 ${from} ≈ ${d.rate.toFixed(4)} ${to}`
            + (when ? `　·　${when}` : '');
    } catch (e) { $('cur_result').textContent = '—'; toast(e.message); }
}
// 打字用防抖：停手 350ms 才查，避免每按一鍵就打一次 API
function scheduleConvert() { clearTimeout(curTimer); curTimer = setTimeout(doConvert, 350); }

$('s_currency').addEventListener('click', () => {   // 開視窗
    fillCur($('cur_from'), cfg.cur_from); fillCur($('cur_to'), cfg.cur_to);
    $('cur_amount').value = cfg.cur_amount || '1';
    $('currencyModal').classList.remove('hidden');
    doConvert();                                    // 一開就先算一次
});
$('cur_swap').addEventListener('click', () => {     // 對調兩幣別
    const a = $('cur_from').value; $('cur_from').value = $('cur_to').value;
    $('cur_to').value = a; doConvert();
});
$('cur_from').addEventListener('change', doConvert);
$('cur_to').addEventListener('change', doConvert);
$('cur_amount').addEventListener('input', scheduleConvert);   // 打字即時算
$('cur_close').addEventListener('click', () => $('currencyModal').classList.add('hidden'));

doConvert() 每一段在幹嘛
這幾行在做什麼

步驟三：逐段看懂 (2)

為什麼打字要「防抖」

金額框綁的是 scheduleConvert 而不是 doConvert。差別在：如果你打「1000」，會依序輸入 1→10→100→1000，四個字四次。若每一鍵都馬上查匯率，就打了 4 次 API，又慢又浪費。

防抖（debounce）的做法：每按一鍵就把上一個計時器取消、重設一個「350 毫秒後才查」。只要你手還在動就一直重設，等你停手那一刻才真正查一次。

✅ 相對地，「換幣別」是一次性動作（不會連續狂點），所以直接綁 doConvert 立刻算，不用防抖。
💡 $() 是專案裡的小幫手，等同 document.getElementById()；toast() 是彈出小提示的小幫手，前面章節已寫好。

🔧 卡住了？本章常見的四個坑

對症下藥
① 查台幣一直失敗：多半是不小心把 Frankfurter 當成主來源——它沒有 TWD。確認主來源是 open.er-api.com。
② 回「查不到匯率」：幣別代碼打錯（如把日圓打成 JYP）。用清單裡的三碼代碼；後端已幫你 .upper() 轉大寫。
③ 打字時卡卡或狂閃：金額框忘了用防抖，每一鍵都在查。確認綁的是 scheduleConvert 而不是 doConvert。
④ 按鈕沒反應：按 F12 看 Console。404＝後端沒加 /api/currency 或沒重啟；cur_from is null＝index.html 的 id 沒對上。

動手練習

改一點點，感受差異
  • 在 CURRENCIES 清單加一個新幣別，例如 { code: 'GBP', label: 'GBP 英鎊' }，重整後看下拉多一個選項、也能換算。
  • 把 result.toFixed(2) 的 2 改成 4，換一次越南盾（VND），體會「大額用 2 位、小額用 4 位」的差別。
  • 故意把主來源網址打錯一個字（如 open.er-apiX.com），重啟後換一組沒有 TWD 的（例如 USD→EUR），觀察它自動退到 Frankfurter 仍算得出來——親眼見證「容錯」。
