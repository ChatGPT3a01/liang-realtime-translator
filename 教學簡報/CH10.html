先回顧：我們現在有什麼

站在前面章節的成果上，往上加一層

到上一章為止，你的 App 已經會：打字翻譯、語音翻譯、即時口譯。這些都是「你一個人拿著手機」在用。

但真實出國情境常常是——你和店員、路人面對面，你講中文、他講外文，兩個人要輪流看同一支手機。這一章就把這個場景做出來。

🧩 本章只加一塊：一個「面對面」畫面（上下對開）＋一顆「切換模式」按鈕。後端完全不動，全部沿用前面寫好的翻譯與語音。

這一章要做出什麼

兩人隔著手機對話

把手機平放桌上，畫面上下對開：對面的人看倒過來的上半、你看正的下半。各自講各自的語言，翻譯顯示在對方那一側——他就能正著讀你的話。

🎁 本章小成品

一個真正「面對面」的翻譯機：適合出國問路、店家溝通，一支手機兩個人用。

設計概念：為什麼上半要翻過來

手機平放，兩人各據一邊
┌─────────────┐
│  對面的人看   │  ← 上半，旋轉 180°（給對面的人正著看）
│ （翻轉的文字）│
├─────────────┤
│   你看的     │  ← 下半，正常方向（給你看）
│（正常的文字）│
└─────────────┘
手機平放桌上，你在這邊、對方在那邊

關鍵直覺：手機平放時，坐你對面的人是「上下顛倒」看螢幕的。所以上半整塊轉 180°，對他來說就變正的了。

先看全貌：一次對話中間發生什麼事

看懂這條路，後面每一步就知道自己在做哪一段
  1. 你按頂部「👤 單人 / 👥 面對面」鈕 → 切到面對面畫面，手機平放桌上
  2. 你在下半按「🎤 說話」講中文 → 講完再按一下停止
  3. 前端把整段中文丟給後端翻成上半的語言（例如英文）
  4. 翻譯結果顯示在上半那一框（翻轉的，對面的人正著看）＋朗讀出來
  5. 對面的人在上半按「🎤 說話」講英文 → 翻成中文顯示在下半給你看
👉 本章就是照著這條路，把「上下兩塊畫面 + 各自麥克風 + 翻到對側顯示 + 模式切換」做出來。前面章節已經寫好的 translate()、Recognizer、speak() 直接重用。

白話觀念：兩個關鍵詞

rotate(180deg) 與「重用」
🔄

rotate(180deg)

CSS 的一行指令，把一整塊畫面「轉半圈」。不是重寫程式，只是把已經排好的東西「掉頭」給對面的人看。

♻️

重用（不重寫）

翻譯、語音辨識、朗讀在前面章節都做好了。面對面模式只是把它們「接兩份」——上一份、下一份，各接各的麥克風。

💡 這一章幾乎沒有全新的技術，重點是「怎麼把既有零件排成上下對開、並讓兩邊互相顯示」。看懂結構比背程式碼重要。

步驟二：逐段看懂 + 為什麼只轉上半

一行 transform 的魔法

.face-panel { flex: 1 }：上下兩塊各佔一半高度，畫面剛好對開。

.face-panel.top { transform: rotate(180deg) }：只有上半整塊順時針轉半圈。文字、選單、按鈕全部跟著掉頭——對坐你對面、上下顛倒看螢幕的人來說，剛好變正的。

下半不轉，因為那是給你（拿手機這邊的人）看的，本來就正的。

✅ 就這麼簡單：不用做兩套版面、不用鏡像文字，一行 rotate(180deg) 就把「對面視角」解決了。這是 CSS 最漂亮的小技巧之一。
🔧 卡住了？　若上半沒翻、或整頁一起翻了 → 確認 rotate(180deg) 是加在 .face-panel.top（只有上半那塊），不是加在 #faceView（整個外框）。

📂 打開這個檔案：static/js/app.js，把它貼在檔案裡「模式切換」那段的前面（也就是即時口譯相關函式的後面）。

💬 對 AI 這樣說：

幫我寫 makeFaceSide(langSelId, resultBoxId, micBtnId, getTargetId)：
用現成的 Recognizer 監聽這一塊的麥克風按鈕；
講完停止後，把整段用 translate() 翻成對面的語言，

然後呼叫兩次：下塊 f_micBottom 翻到上塊、上塊 f_micTop 翻到下塊。

步驟三：AI 會給你這段（貼上去）

貼進 static/js/app.js
// ===== 面對面模式 =====
function makeFaceSide(langSelId, resultBoxId, micBtnId, getTargetId) {
  const box = $(resultBoxId), btn = $(micBtnId);
  const rec = new Recognizer({
    bcp: byId($(langSelId).value).bcp,
    onInterim: (t) => setResult(box, t, true),   // 邊講邊顯示（暫時的）
    onDone: async (t) => {
      const src = $(langSelId).value, tgt = getTargetId();
      try {
        const out = await translate(t, src, tgt);
        // 顯示在「對面那一側」的框
        const otherBox = (resultBoxId === 'f_resultBottom')
          ? $('f_resultTop') : $('f_resultBottom');
        setResult(otherBox, out);
        speak(out, byId(tgt).bcp);            // 念出來給對面聽
        pushHistory(t, out);
      } catch (e) { toast(e.message); }
    },
    onState: (on) => btn.classList.toggle('listening', on),
  });
  btn.addEventListener('click', () => { rec.bcp = byId($(langSelId).value).bcp; rec.start(); });
  return rec;
}
// 下方(你)講 → 翻成上方語言 → 顯示在上方(翻轉給對面看)
makeFaceSide('f_langBottom', 'f_resultBottom', 'f_micBottom', () => $('f_langTop').value);
// 上方(對面)講 → 翻成下方語言 → 顯示在下方(給你看)
makeFaceSide('f_langTop', 'f_resultTop', 'f_micTop', () => $('f_langBottom').value);

步驟三：逐段看懂

這段每一行在幹嘛
這幾行在做什麼
new Recognizer({...})沿用前面章節的語音辨識器，bcp 是這塊選的語言的辨識代碼
onInterim → setResult(box, t, true)你還在講的時候，把「暫時的字」先顯示在自己這塊，有回饋感
onDone: async (t)你講完按停後才進來，拿到整段話 t
translate(t, src, tgt)把整段翻成對面的語言（getTargetId() 取的是對面選單的值）
otherBox = ... ? f_resultTop : f_resultBottom本章靈魂：翻好的字塞進「對面那一框」，不是自己這框
speak(out, byId(tgt).bcp)用對面語言的發音把翻譯念出來

步驟三 · 重點拆解：為什麼「翻到對面那一框」

整章最容易搞混的地方

直覺上你會以為「翻譯結果顯示在我這邊」。但面對面情境剛好相反——

  • 你在下半講中文，是要給對面的人看懂，所以英文翻譯要出現在上半（他正著讀的那塊）。
  • 對面在上半講英文，是要給你看懂，所以中文翻譯出現在下半。
🔁 所以程式碼用 otherBox：講話的是這一塊，結果卻寫到「另一塊」。這一個小反轉，就是面對面模式能運作的關鍵。
✅ 順帶好處：onInterim 把「你正在講的暫時字」顯示在自己這塊，翻好的成品送到對面——講者和聽者各看各的，不會打架。

步驟三 · 巧妙處：一個函式，接兩份麥克風

為什麼呼叫兩次就好

makeFaceSide() 把「一邊的完整行為」包成一個函式，只要告訴它四件事：用哪個語言選單、聽哪顆麥克風、結果框是哪個、對面語言去哪拿。

於是上下兩邊只差參數不同，同一段程式呼叫兩次就完成：

呼叫誰講翻到哪
第一次（bottom）你（下半）上半，給對面
第二次（top）對面（上半）下半，給你
💡 這叫「工廠函式」——寫一次邏輯、用參數生出兩個一模一樣但各接各線路的實例。改一次兩邊一起改，不用複製貼上兩份。

// ===== 模式切換 =====
let mode = 'single';
$('modeBtn').addEventListener('click', () => {
  if (liveOn) stopLive();                         // 若還在即時口譯，先停掉
  mode = mode === 'single' ? 'face' : 'single';   // 兩種模式來回切
  $('singleView').classList.toggle('hidden', mode !== 'single');
  $('faceView').classList.toggle('hidden', mode !== 'face');
  $('modeBtn').textContent = mode === 'single' ? '👤 單人' : '👥 面對面';
});
