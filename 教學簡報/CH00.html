什麼是 Vibe Coding？

用「說的」把軟體做出來

Vibe Coding（氛圍編程）是近年因 AI 崛起而流行的做法：你不用自己一行一行寫程式，而是用自然語言描述你想要什麼，讓 AI 幫你把程式碼寫出來。

你的工作從「背語法、記指令」，變成「把需求講清楚、看結果、再調整」。就像跟一位很厲害的工程師夥伴合作，你負責出點子與方向，它負責動手。

🗝️ 關鍵能力不是打字，而是「怎麼把腦中的東西，講到 AI 聽得懂、做得對」。這就是整堂課要練的功夫。

Vibe Coding 三個心法

先記住，後面每一章都會用到
🎯

講清楚「要什麼」

目標、輸入、輸出、限制講明白。含糊的需求會得到含糊的結果。

🔁

小步快跑、逐步加

一次只做一個功能，做好了再加下一個，不要一次要一大包。

🔍

看結果、回饋、修正

跑跑看、哪裡不對就告訴 AI，讓它改。對話式地把東西磨好。

每一章，都有能跑的小成品

功能一個一個疊上去，隨時都能用

這堂課不是講完理論才動手。每一章結束，你都會得到一個「可以使用的小成品」——先能翻譯文字，再加語音，再加拍照…功能一層一層疊上去，每一層都能實際打開來玩。

🎁 可用的小成品

每章結尾都有這個綠色框，告訴你「現在你的 App 已經能做什麼了」。到課程最後，這些小成品就疊成一個完整的【亮言即時翻譯】App。

這堂課怎麼上？

每個步驟 = 一段可複製的提示詞 + 產出的程式碼
  1. 老師先講「這一步我們要做什麼、為什麼」。
  2. 投影片上給你一段可以直接複製的提示詞，或一個填欄位就生成程式碼的生成器。
  3. 你把提示詞貼給 AI（Claude／ChatGPT 等），得到程式碼。
  4. 把程式碼放到專案對應的位置，執行看看。
  5. 對照投影片的「預期結果」，不一樣就照「除錯提示」處理。

提示詞長什麼樣子？搶先看

這就是你之後會一直用的東西

比方說，要生出網站骨架，你會用到類似這樣的提示詞：

💬 提示詞範例
請用 Python Flask 幫我建立一個最小的網站專案，包含：
1. app.py：啟動一個網頁伺服器，首頁回傳「Hello 亮言」。
2. requirements.txt：列出需要的套件。
3. 告訴我如何在本機執行、用瀏覽器打開看結果。
請一步一步說明，我完全沒有寫過程式。

看到重點了嗎？講清楚要什麼、列出輸出、說明我的程度——這就是好提示詞。

會不會花錢？

跟著課程做，基本上零成本

這堂課刻意選用「免費」或「免費額度」的服務：

  • 天氣、匯率用免金鑰的公開 API，完全免費。
  • Gemini、Tavily 有免費額度，課堂用量遠遠用不完。
  • GitHub、Render 都有免費方案，足夠讓 App 上線。
⚠️ 免費額度仍有上限。若你把 App 開放給很多人用、或大量測試，可能需要升級付費方案。課堂練習不會遇到。

重要觀念：金鑰要保密

這是每個做 App 的人都要有的資安意識

API 金鑰就像你家鑰匙——拿到的人可以用你的額度、你的帳單。所以：

  • ❌ 不要把金鑰直接寫死在程式碼裡、上傳到 GitHub。
  • ✅ 用「環境變數」的方式存放（課程會教）。
  • ✅ 不小心外流了，就到後台把它「重新產生（regenerate）」作廢舊的。
🔐 我們的做法：金鑰放在伺服器的環境變數，或 App 的「設定」裡，絕不寫進會公開的程式碼。

卡關了怎麼辦？

這才是真正學會的地方
  1. 先看投影片的「預期結果 / 除錯提示」，多半有寫。
  2. 把完整的錯誤訊息複製，貼給 AI 問：「我遇到這個錯誤，怎麼解？」
  3. 一次只改一個地方，改完再跑，才知道是哪裡的問題。
  4. 真的卡住，回到上一個「能動」的版本，再小步前進。
💪 記住：出錯是正常的，工程師每天都在解錯。會解錯，你就真的會了。
