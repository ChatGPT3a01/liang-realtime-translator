先回顧：我們現在有什麼

站在 CH04 的成果上

到上一章，你有一個能跑的 Flask 後端：首頁回文字、/api/health 回 JSON。但打開瀏覽器只有一行字，還不像 App。

這章把「臉」做出來。之後 CH06 再讓按鈕真的會翻譯——一層一層疊。

前端三兄弟

畫面是這三個一起做出來的
🧱

HTML

結構：有哪些元素（選單、按鈕、框）。像房子的「隔間」。

🎨

CSS

樣式：顏色、大小、圓角、排版。像房子的「裝潢」。

⚡

JavaScript

行為：按了會發生什麼事。像房子的「電路」。

Flask 怎麼給前端？

templates 放頁面、static 放樣式與腳本
📁 資料夾結構（要建這些）
專案/
├─ app.py
├─ templates/
│   └─ index.html    ← 主頁面
└─ static/
    ├─ css/style.css  ← 樣式
    └─ js/app.js      ← 前端邏輯

Flask 有約定：templates 放 HTML 頁面、static 放 css/js/圖片。照這個結構放，Flask 自動找得到。

先把資料夾建好

在 VS Code 左側新增
  1. 在專案下建 templates 資料夾，裡面新增 index.html。
  2. 建 static 資料夾，裡面再建 css 與 js 兩個子資料夾。
  3. 在 css 建 style.css、在 js 建 app.js。
💡 VS Code 左側檔案總管：對資料夾按右鍵 → New File / New Folder。

逐段看懂 (1)：head

頁面的「設定區」

<meta charset="UTF-8">：用萬國碼，中文才不會變亂碼。

<meta name="viewport" ...>：手機上正確縮放，這是「手機優先」的關鍵一行。

<link rel="stylesheet" href="/static/css/style.css">：載入我們的樣式檔（Flask 會從 static 提供）。

逐段看懂 (2)：語言列與結果框

body 的上半

<select id="langA"> / langB：兩個語言下拉選單，中間 swap 鈕用來對調。選項先留空，等等用 JS 動態填。

<div class="result" id="result">：翻譯結果要顯示的框。id 是給 JS 抓取用的「名牌」。

🏷️ 記住這些 id（langA、langB、result、mic、text、send）——CH06 的 JS 就是靠它們找到元素。

逐段看懂 (3)：麥克風與輸入框

body 的下半

<button id="mic">：麥克風按鈕，CH07 會接語音辨識。

<textarea id="text">：讓使用者打字的輸入框；<button id="send"> 是翻譯鈕。

<script src="/static/js/app.js">：放在 body 最後，載入前端邏輯（放最後是為了確保上面元素都存在了）。

style.css（重點）

深紫漸層 + 金色重點
/* static/css/style.css */
:root{ --bg1:#1a1030; --bg2:#2a1a4a; --gold:#f5c451; --text:#f3eefb; }
*{ box-sizing:border-box; margin:0; }
body{ font-family:'Noto Sans TC',sans-serif; color:var(--text);

逐段看懂 CSS (1)

變數與背景

:root{ --gold:#f5c451; ... }：定義「顏色變數」。之後 var(--gold) 就能重複用，改一次全站生效。

* { box-sizing:border-box; }：讓寬高計算含邊框，排版比較不會爆版。

background:linear-gradient(...)：深紫漸層背景，就是這行讓畫面有質感。

逐段看懂 CSS (2)

卡片與按鈕

.result{ background:rgba(255,255,255,.07); border-radius:16px; }：半透明白 + 圓角，做出「毛玻璃卡片」感。

.mic-btn{ background:var(--gold); ... }：金色大按鈕，好按、醒目。

display:flex; gap:10px;：讓語言列、輸入列的元素橫向並排並留間距。flex 是排版最常用的工具。

CSS 變數的威力

改一個色，全站換裝

試著把 --gold:#f5c451 改成 --gold:#4ac0ff（藍），存檔、重整——所有金色的地方（標題、按鈕）一次全變藍。

🎨 這就是為什麼我們把顏色抽成變數：想換品牌色，改一行就好。CH19 的主題生成器也是靠這個。

app.js（骨架：先分好 10 區）

整支檔案先畫好格子，之後每章填一格
// static/js/app.js —— 整支檔案先分成 10 區，之後每一章填其中一區
// ===== 1. 設定與語言 =====   （語言清單、預設設定，這章先放一點）
// ===== 2. 翻譯 =====          （CH06 會填）
// ===== 3. 語音辨識 STT =====  （CH07 會填）
// ===== 4. 朗讀 TTS =====      （CH08 會填）
// ===== 5. 即時語音 =====      （CH09 會填）
// ===== 6. 面對面 =====        （CH10 會填）
// ===== 7. 拍照 / 檔案 =====   （CH11–12 會填）
// ===== 8. 匯率 / 天氣 / 助手 =====（CH13–15 會填）
// ===== 9. 設定面板 / 密碼 / 換風格 =====（CH16–17 會填）
// ===== 10. 啟動 =====
async function checkHealth(){
  const res = await fetch('/api/health');
  const data = await res.json();
  console.log('後端狀態：', data);   // 應印出 {ok: true}
}
checkHealth();

逐段看懂 app.js

前端跟後端「握手」

fetch('/api/health')：向後端要健康狀態。

await res.json()：等回覆並解析成物件。

console.log(...)：印到瀏覽器 Console（開發用，使用者看不到）。

checkHealth()：頁面一載入就執行，馬上驗證前後端通不通。

🗺️ 你的 app.js 骨架地圖

先看懂這張，之後每章都不會迷路

剛剛那 10 個 // ===== 區 ===== 就是你整支 App 的「格子」。之後每一章，只會挑其中一格把程式碼填進去，其他格不動。

區哪一章填狀態
1 設定與語言CH05（現在）✅ 已放一點
2 翻譯CH06⬜ 待填
3 語音辨識 STTCH07⬜ 待填
4~9 各功能CH08~CH17⬜ 待填
10 啟動CH05（現在）✅ checkHealth
👉 以後每章的口訣都一樣：「打開 app.js，找到 // ===== X ===== 這一區，把程式碼貼在它下面。」不用再煩惱貼哪裡。

💾 每章都能「存檔點」下載

做壞了、跟丟了，隨時抓乾淨的檔重來

從下一章起，每一章的開頭和結尾都會有一顆下載鈕：

  • 開頭「上一章完成檔」：萬一你上一章做壞了，直接下載這個當起點，接著做本章。
  • 結尾「本章完成檔」：本章做完的完整專案，拿來跟你自己的對照，或存起來。
🎮 就像遊戲的「存檔點」——你永遠有一個乾淨、能跑的版本可以回去，不怕卡關。

怎麼看 Console？

F12 是你的好朋友
  1. 在瀏覽器按 F12（或右鍵 → 檢查）。
  2. 切到「Console」分頁。
  3. 重整頁面，看有沒有印出 後端狀態： {ok: true}。
🔍 之後任何前端問題，第一步都是打開 Console 看紅字。這是除錯的起點。

更新 app.py 提供頁面

用 render_template 回傳 index.html
# 把首頁改成回傳 index.html
from flask import Flask, jsonify, render_template

@app.route('/')
def index():
    return render_template('index.html')

記得把最上面的 import 補上 render_template；否則會報「render_template is not defined」。

把語言選單填起來

用 JS 動態產生選項
// app.js — 語言清單
const LANGS = [
  {id:'zh-TW', label:'繁體中文'}, {id:'en', label:'English'},
  {id:'ja', label:'日本語'}, {id:'ko', label:'한국어'},
];
function fill(sel, val){
  sel.innerHTML = LANGS.map(l => `<option value="${l.id}">${l.label}</option>`).join('');
  sel.value = val;
}
fill(document.getElementById('langA'),'zh-TW');
fill(document.getElementById('langB'),'en');

逐段看懂 fill()

動態塞選項

LANGS.map(l => ...)：把每個語言變成一段 <option>。

sel.innerHTML = ...：把這些選項塞進選單。

sel.value = val：設定預設選哪個。

💡 之後要加語言，只要在 LANGS 陣列多加一行，兩個選單自動都有。這叫「單一資料來源」。

除錯 1：畫面沒有樣式

背景是白的、很陽春

症狀：畫面出現但沒有深紫背景、沒有圓角，像純文字。

原因：CSS 沒被載入。檢查：

  • style.css 是否在 static/css/ 下？
  • index.html 的 link 路徑是否 /static/css/style.css？
  • F12 → Network 看 style.css 是不是 404。

除錯 2：Console 紅字

JS 出錯

langA is not defined

元素還沒載入就用它。確認 script 放在 body 最後。

Failed to fetch /api/health

後端沒開或路由不對，確認 app.py 有 health 路由、伺服器在跑。

選單沒有選項

fill() 沒跑到，確認 id 是 langA/langB、程式碼有貼進 app.js。

404 index.html

確認 app.py 改用 render_template，且檔案在 templates/。

手機優先設計

為什麼一切都「直向、大顆」

我們的按鈕都做很大、元素都直向排列——因為這是手機拿在手上用的 App。大按鈕好按、直向適合單手滑。

📱「手機優先」不只是縮小電腦版，而是一開始就為小螢幕、手指操作而設計。
