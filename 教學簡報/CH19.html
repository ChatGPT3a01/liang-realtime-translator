這個生成器會產出

完整、可直接執行的專案
你的專案.zip（完整可跑）
├─ app.py              ← 後端主程式（完整版）
├─ providers.py        ← 翻譯供應商抽象層
├─ requirements.txt    ← 套件清單
├─ render.yaml         ← 部署設定
├─ .env                ← 你填的金鑰（自動注入）
├─ .gitignore
├─ templates/index.html   ← 完整前端
└─ static/（app.js / style.css / sw.js / manifest.json）
✅ 這不是骨架——是和老師示範完全相同的正式版程式碼，解壓後裝套件就能直接跑。
🔑 下載版的預設密碼是 1234（示範用，不是老師的密碼）。第一次進站用它登入，請務必用 CH17 的 SHA-256 生成器換成你自己的密碼再部署。

🎁 整合專案生成器

填一填，按「打包下載」
⚙️ 產生你的專案（勾一勾、填金鑰）
找不到你要的語言？可直接寫語言名稱（例如「德語」），或 👉 點此查語言代碼 再填進上面。
還沒有？👉 點此免費申請 Gemini 金鑰。可留空，但留空的話，下載後要到 App 右上角 ⚙️ 設定、或部署時在 Render 環境變數 補上，否則不能翻譯。
還沒有？👉 點此免費申請 Tavily 金鑰。留空也行，旅遊助手會改用 AI 既有知識回答。
三種身分都支援:① 用 Gemini → 這欄留空即可;② 想讓「伺服器共用一把 OpenAI 金鑰」→ 填這裡,會寫進 .env;③ 每位使用者各自帶自己的 OpenAI 金鑰 → 下載後在 App 的 ⚙️ 設定 面板填(存自己瀏覽器)。
若你前面章節有自己疊出檔案、打包成 zip，可以上傳留個紀念——你的努力有被收下。不過下載的仍是老師驗證過的完整正式版，確保你一定跑得起來。
勾選、填金鑰後按「產生預覽」看內容，或直接「打包下載 .zip」…
下載的 zip 是完整正式版，已把你的金鑰寫進 .env。填「品牌名稱」的話，App 標題／頁首／PWA 名稱／密碼閘門裡的「亮言」會全部換成你的品牌。解壓 → 裝套件 → 執行，立刻能跑。

下載後怎麼用？

三步驟直接啟動
  1. 解壓縮 zip，用 VS Code 打開資料夾。
  2. 終端機 pip install -r requirements.txt 裝好套件。
  3. python app.py → 開瀏覽器 http://127.0.0.1:5001 → 你的 App 就跑起來了！
✅ 這就是老師示範的同一份正式程式。要上線，照 CH18 上傳 GitHub → Render 即可。
