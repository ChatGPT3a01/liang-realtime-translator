什麼是 STT？

Speech To Text，語音轉文字

STT 把「聲音」變成「文字」。瀏覽器提供了 Web Speech API，不用裝任何東西、不用金鑰，就能做語音辨識。

⚠️ 這個 API 在 Chrome / Edge 支援最好；部分瀏覽器不支援，我們會加「不支援就用打字」的後備。

語言碼：BCP-47

告訴辨識器你講哪國話

語音辨識要指定語言碼，例如繁中是 zh-TW、英文 en-US、日文 ja-JP。這叫 BCP-47 格式。

我們把每個語言的「顯示名稱、給模型的英文名、BCP-47 碼」整理成一個清單，一次管好。本章最後有生成器幫你產生。

提示詞：做語音辨識

講清楚「逐句」的行為
用瀏覽器 Web Speech API 幫我做語音辨識：
- 點麥克風開始聽，即時把逐字稿顯示在結果框。
- 再點一下停止，把整段文字交出去做翻譯。
- 處理麥克風權限被拒、瀏覽器不支援的情況。
給我一個 Recognizer class 與接到按鈕的程式碼，我是新手。

Recognizer（重點）

包一個好用的辨識器
// app.js
const SR = window.SpeechRecognition || window.webkitSpeechRecognition;
class Recognizer{
  constructor(bcp, onDone){ this.bcp=bcp; this.onDone=onDone; this.active=false; this.buffer=''; }
  start(){
    if(!SR){ alert('此瀏覽器不支援語音辨識，請改用打字'); return; }
    if(this.active){ this.stop(); return; }   // 再點一下 = 停止並翻譯
    this.buffer=''; const rec=new SR();
    rec.lang=this.bcp; rec.interimResults=true; rec.continuous=true;
    rec.onresult=(e)=>{ let s=''; for(const r of e.results) s+=r[0].transcript;
      result.textContent=s; this.buffer=s; };
    rec.onend=()=>{ if(this.active) rec.start(); };  // 自動續聽
    this.rec=rec; this.active=true; rec.start();
  }
  stop(){ this.active=false; this.rec && this.rec.stop();
    const t=this.buffer.trim(); if(t) this.onDone(t); }
}

為什麼「說完再翻」？

逐句模式的用意

我們讓使用者「說完、按停」才整段翻譯，而不是邊說邊翻。這樣：

  • 翻譯更完整、更準（有完整句子）。
  • 之後加「朗讀」時，朗讀不會佔用麥克風、造成混亂。
🎯 這就是課程名稱裡「逐句」的意思：一句一句，穩穩來。

實測 + 權限

第一次會問麥克風權限

重整頁面，點麥克風。瀏覽器會跳出「允許使用麥克風？」→ 按允許。說一句中文，再點一下麥克風停止。

✅ 逐字稿會即時顯示，停止後整段被翻成目標語言。你用說的就翻好了！

常見問題

沒反應先看這

完全沒反應？

用 Chrome/Edge 試；本機用 http://127.0.0.1 沒問題，正式上線需 HTTPS。

權限被拒？

網址列左邊鎖頭 → 重設麥克風權限 → 重整。

辨識成別的語言？

確認 rec.lang 用對 BCP-47，且對應你選的來源語言。

手機上不穩？

正式版可搭配後面的 Gemini Live 即時模式（CH09）。
