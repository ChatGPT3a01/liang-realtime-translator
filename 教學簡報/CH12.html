先回顧：我們現在有什麼

站在 CH11 拍照翻譯的成果上再加一層

上一章我們讓 App 學會「看照片」：按「📷 拍照翻譯」拍一張，Gemini 就給你「摘要＋翻譯」。核心是後端那個 providers.py 裡的 analyze() 函式。

這一章我們不重寫 analyze()，只是換一種「餵資料進去」的方式——讓使用者能直接上傳一個檔案：圖片、PDF、文字檔都行。

🧩 本章只加一塊：一顆「📎 上傳檔案」按鈕 + 一道新的門 /api/file。analyze() 幾乎原封不動沿用。

這一章要做出什麼

選一個檔 → 摘要 + 翻譯

按「📎 上傳檔案」，從手機或電腦裡挑一個檔案：外文菜單照片、一份英文PDF 說明書、或一個 txt 文字檔。App 讀懂它，用你的語言摘要重點，再附上完整翻譯。

🎁 本章小成品

把外文 PDF、圖片或文字檔丟進來，就能得到「摘要＋翻譯」——不用先自己開檔、複製貼上。

先看全貌：選一個檔，中間發生什麼事

看懂這條路，後面每一步就知道自己在做哪一段
  1. 使用者按「📎 上傳檔案」→ 跳出「選檔案」視窗，挑一個檔
  2. 前端判斷：圖片就先縮小；PDF／文字檔就原樣
  3. 前端用 FormData 把檔案送到後端的 /api/file
  4. 後端看檔案類型分流：文字→讀文字、PDF→當 PDF、圖片→當影像
  5. 三條路最後都交給同一個 providers.analyze() 去讀
  6. 拿回 {summary, translation}，前端顯示在畫面上
👉 重點只有兩個新東西：前端「用 FormData 送檔」、後端「依類型分流」。其餘都跟 CH11 一樣。

白話觀念①：這次為什麼用「FormData」

跟 CH11 拍照的送法不一樣

CH11 拍照時，我們把照片轉成一長串文字（dataURL），用 JSON 送。但這章要送的可能是 PDF，把整個 PDF 塞進 JSON 又肥又慢。

所以改用 FormData——它是瀏覽器內建、專門用來「像交表單一樣夾帶檔案」的工具，可以把原始檔案直接夾著送，不用轉成文字。

CH11 拍照（JSON）CH12 上傳（FormData）
送什麼照片轉成的文字 dataURL原始檔案本身
後端怎麼收request.get_json()request.files.get('file')
要不要設 Content-Type要設 application/json不用設，瀏覽器自動處理

白話觀念②：三種檔案，三條路

圖片、PDF、文字，各走各的分流
檔案類型怎麼處理交給 analyze() 的參數
圖片 jpg/png…當影像丟給 AI（同拍照）file_bytes + mime_type
PDF當 application/pdf 原生讀取file_bytes + mime_type='application/pdf'
文字檔 txt/md/csv直接讀出裡面的文字text（純文字）
📌 PDF 有個限制：只有 Gemini 能原生「讀」PDF。若使用者在設定裡選了別家（例如 OpenAI 相容），後端會自動改用 Gemini 來讀這份 PDF，並回一句提示。這點下面步驟二會再說。

白話觀念③：這章為什麼「幾乎不用寫 AI 程式」

CH11 的 analyze() 早就準備好了

# app.py — 檔案大小上限與可接受的圖片副檔名
MAX_FILE_BYTES = 15 * 1024 * 1024   # 15MB
IMAGE_EXTS = {'jpg','jpeg','png','webp','heic','heif','gif'}

@app.route('/api/file', methods=['POST'])
def api_file():
    f = request.files.get('file')                 # FormData 裡的檔案
    target = request.form.get('target') or 'Traditional Chinese (Taiwan)'
    if not f: return jsonify({"ok":False,"error":"no file"}), 400
    filename = f.filename or 'upload'
    raw = f.read()                                 # 讀成 bytes
    if not raw: return jsonify({"ok":False,"error":"空白檔案"}), 400
    if len(raw) > MAX_FILE_BYTES:
        return jsonify({"ok":False,"error":"檔案過大（上限 15MB）"}), 400
    mime = (f.mimetype or '').lower()
    ext = filename.rsplit('.',1)[-1].lower() if '.' in filename else ''

    if mime=='text/plain' or ext in ('txt','md','csv'):       # ① 文字檔
        text = raw.decode('utf-8', errors='replace')
        result, status = _run_analyze(pc, target, text=text)
    elif mime=='application/pdf' or ext=='pdf':               # ② PDF → 強制 Gemini
        result, status = _run_analyze(pc, target, force_gemini=True,
                                      file_bytes=raw, mime_type='application/pdf')
    elif mime.startswith('image/') or ext in IMAGE_EXTS:      # ③ 圖片
        if not mime.startswith('image/'): mime = 'image/jpeg'
        result, status = _run_analyze(pc, target, file_bytes=raw, mime_type=mime)
    else:
        return jsonify({"ok":False,"error":f"不支援的檔案類型：{mime or ext}"}), 400

    result.setdefault("filename", filename)        # 把檔名一起回傳
    return jsonify(result), status

💡 pc 是「使用者選了哪家 AI」的設定；_run_analyze() 是個小幫手，負責挑金鑰、呼叫 analyze()、包好錯誤——下一張說明。

這幾行在做什麼

① 文字：raw.decode('utf-8')把 bytes 轉回可讀文字，用 text= 送給 analyze()
② PDF：force_gemini=TruePDF 只有 Gemini 能讀 → 強制用 Gemini（下一張細講）

步驟一 · 重點：PDF 為什麼要「強制 Gemini」

force_gemini=True 的用意

圖片幾乎每家 AI 都看得懂，但直接讀 PDF 是 Gemini 的強項——它能原生把 PDF 當一份文件讀進去。若使用者在設定裡選了別家（例如 OpenAI 相容供應商），碰到 PDF 就會讀不了。

✅ 解法：遇到 PDF 時傳 force_gemini=True，後端就自動改用 Gemini 來讀這一份，並在回傳裡附一句 note：「PDF 不支援所選供應商，已自動改用 Gemini」。使用者完全不用管，照樣拿到翻譯。
🔧 卡住了？　PDF 一直失敗，多半是：① 伺服器沒有 Gemini 金鑰（就算選別家，PDF 也要靠它）→ 到 .env 補 GEMINI_API_KEY；② 檔案超過 15MB → 先壓縮或拆頁。

// 按「📎 上傳檔案」→ 點一下那個藏起來的檔案框
$('s_file').addEventListener('click', () => $('fileInput').click());

$('fileInput').addEventListener('change', async (e) => {
  const file = e.target.files && e.target.files[0];
  e.target.value = '';                       // 清掉，下次選同一檔也能觸發
  if (!file) return;
  openVision('📎 檔案翻譯');                  // 打開結果彈窗
  try {
    const fd = new FormData();               // ← 關鍵：用表單方式夾檔
    fd.append('target', byId($('vision_target').value).name);
    if (file.type.startsWith('image/')) {    // 圖片：先縮圖再送
      const dataURL = await fileToDownscaledDataURL(file);
      fd.append('file', dataURLToBlob(dataURL), 'upload.jpg');
    } else {                                 // PDF / 文字檔：原檔直接送
      fd.append('file', file);
    }
    $('visionStatus').textContent = '上傳與翻譯中…（檔案越大越久）';
    const res = await fetch('/api/file', { method:'POST', body: fd });  // 不用設 Content-Type
    const data = await res.json();
    if (!data.ok) throw new Error(data.error || '翻譯失敗');
    renderVision(data);                      // 顯示摘要 + 翻譯
  } catch (err) { $('visionStatus').textContent = '❌ ' + err.message; }
});

為什麼圖片要縮、PDF 卻不縮？

兩種檔的取捨不一樣

小補充：一個檔案在網路上怎麼「飛過去」

multipart／form-data 的直覺
✉️

FormData ＝ 信封

把「檔案」和「target 語言」一起裝進一個信封。瀏覽器會自動貼好正確的郵票（Content-Type），你不用手動設。

📬

後端 ＝ 收件人

Flask 用 request.files 取信封裡的檔案、request.form 取一般欄位。跟 JSON 走的 request.get_json() 是兩套不同信箱。

💡 記住一句話就好：要夾檔案 → 用 FormData；只送文字資料 → 用 JSON。這章夾了檔，所以用 FormData。

🔧 卡住了？本章常見的四個坑

對症下藥
① 回「不支援的檔案類型」：你挑的檔不是圖片／PDF／文字。確認副檔名是 .jpg/.png/.pdf/.txt/.md/.csv 之一。
② 回「檔案過大（上限 15MB）」：檔太大。圖片會自動縮，但 PDF 不縮——請先壓縮或拆頁。
③ PDF 一直失敗：多半是伺服器沒有 Gemini 金鑰。就算你選別家，PDF 也要靠 Gemini，.env 要有 GEMINI_API_KEY。
④ 文字檔翻出來是亂碼：檔案不是 UTF-8 編碼。用記事本「另存為 UTF-8」再上傳。

回顧：同一套 analyze()，三種入口

CH11 + CH12 一起看，路更清楚
入口使用者動作送法後端端點
拍照（CH11）📷 開相機拍一張JSON（dataURL）/api/vision
上傳圖片（CH12）📎 選一張圖FormData（縮圖）/api/file
上傳 PDF（CH12）📎 選一份 PDFFormData（原檔）/api/file → Gemini
上傳文字（CH12）📎 選一個 txtFormData（原檔）/api/file
🧩 四個入口，最後都匯流到同一個 providers.analyze()。這就是為什麼加新功能越來越快——底層已經打好地基。

動手練習

  • 把 accept 裡的 application/pdf,.pdf 拿掉，再點按鈕——看「選檔案」視窗是不是就不讓你選 PDF 了（體會 accept 的過濾作用）。
  • 把後端的 MAX_FILE_BYTES 從 15 * 1024 * 1024 改成 1 * 1024 * 1024（1MB），上傳一份大一點的 PDF，看它是不是被擋下並回「檔案過大」。
  • 準備一個中文 txt，把送出的 target 改成 English，看它把中文檔翻成英文。
