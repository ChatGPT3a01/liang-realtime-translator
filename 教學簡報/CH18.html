這一章要做出什麼

從本機走向雲端

把專案上傳 GitHub、在 Render 建立服務、設定金鑰環境變數，得到一個真實網址。手機打開就能用、能裝到主畫面。

🎁 本章小成品

一個公開網址（像 https://你的名字.onrender.com），走到哪打開都能用（密碼把關，只有你和授權的人進得來）。

部署流程總覽

三步驟
你的電腦（程式碼）
   ① 上傳 → GitHub（雲端倉庫）
                 ② 連結 → Render（自動抓程式碼、跑起來）
                              ③ 得到 → 公開網址 🌐
之後：改程式 → push GitHub → Render 自動重新部署

上線前檢查三個檔

確認都在專案根目錄
📦

requirements.txt

列出所有套件，Render 會照它安裝。

🙈

.gitignore

含 .env，確保金鑰不上傳。

⚙️

render.yaml

部署設定（選填但方便）。

requirements.txt（完整）

把用到的都列上
flask
flask-socketio
simple-websocket
python-dotenv
google-genai
requests

render.yaml

告訴 Render 怎麼跑
# render.yaml
services:
  - type: web
    name: liang-translator
    runtime: python
    plan: free
    buildCommand: pip install -r requirements.txt
    startCommand: python app.py
    envVars:
      - key: GEMINI_API_KEY
        sync: false          # 在 Render 儀表板填，不寫進 repo
      - key: TAVILY_API_KEY
        sync: false

① GitHub：建立倉庫

程式碼的雲端家
  1. 登入 github.com，右上 + → New repository。
  2. 取名（例 liang-translator），選 Private（私人），建立。
📸請截圖：GitHub 建立 repository 頁

① 上傳程式碼

兩種方式擇一
🖱️

網頁上傳（新手）

repo 頁 → Add file → Upload files → 把專案檔案拖進去 → Commit。

⌨️

用 Git 指令

git init → add → commit → push。可請 AI 給你完整指令。

⚠️ 上傳前再次確認 .env 沒有被傳上去（.gitignore 要生效）。金鑰外流事小，帳單爆掉事大。

② Render：建立服務

連結 GitHub 專案
  1. 登入 render.com（用 GitHub 帳號）。
  2. New + → Web Service。
  3. 選剛剛的 GitHub 倉庫，按 Connect。
📸請截圖：Render 選擇 GitHub 倉庫的畫面

② 基本設定

填對這幾格
欄位填什麼
RegionSingapore（離台灣近）
Branchmain
Build Commandpip install -r requirements.txt
Start Commandpython app.py
Instance TypeFree

② 設定環境變數（關鍵！）

金鑰放這裡，不進程式碼

往下找到 Environment Variables，把金鑰一組一組加進去：

KeyValue
GEMINI_API_KEY你的 Gemini 金鑰
TAVILY_API_KEY你的 Tavily 金鑰（選填）
🔐 這就是為什麼程式用 os.getenv()——它會來這裡讀，金鑰不出現在程式碼或 GitHub。

② 部署！

按下去，等 2–5 分鐘
  1. 點最下方 Create Web Service。
  2. Render 開始安裝套件、啟動，看 log 跑。
  3. 狀態變 Live（綠色）就上線了！
  4. 頁面上方有你的網址：https://xxx.onrender.com，複製起來。
📸請截圖：Render 狀態 Live + 你的網址

免費方案要知道

休眠與喚醒
  • 免費服務閒置會休眠，第一次連線要等 30–60 秒喚醒，屬正常。
  • 喚醒後就正常快。想常駐可升級付費方案。
🌐 用手機打開你的網址測測看：翻譯、拍照、天氣、匯率、助手都試一輪。還能「加到主畫面」！

之後怎麼更新？

push 就自動部署
改程式 → 上傳/push 到 GitHub
        → Render 偵測到變更
        → 自動重新部署（約 2–5 分鐘）
        → 網址上就是新版

改了看不到？PWA 有快取——把 sw.js 的版本號 +1、手機重新整理兩次。

常見部署問題

上不了線先看這

Build 失敗？

多半是 requirements.txt 少套件或打錯字，看 log 補上。

啟動就掛？

確認 Start Command、app.py 用環境變數 PORT。

功能沒反應？

檢查環境變數金鑰有沒有填、有沒有拼錯。

第一次好慢？

免費方案喚醒中，等 30–60 秒。
