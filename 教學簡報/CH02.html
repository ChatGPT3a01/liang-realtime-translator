先搞懂：什麼是 API 金鑰？

一句話：你和服務之間的「通行證」

當我們的 App 要請 Google 幫忙翻譯，Google 得知道「是誰在用、算誰的帳」。API 金鑰就是那張識別身分的通行證——一長串英數字。

App 每次呼叫服務，都會附上這把金鑰，服務才會回應，並把用量記到你的帳號上。

🔑 所以金鑰 =「你的身分 + 你的帳單」。這也是為什麼下一頁的安全守則超級重要。

金鑰安全三守則

養成好習慣，避免帳單爆掉
🙈

不外流

不要貼到公開的地方、不要寫死在會上傳 GitHub 的程式碼裡。

📦

用環境變數

存在 .env 檔或平台的環境變數，程式用讀取的方式取得。

♻️

外流就重生

不小心外流，立刻到後台 Regenerate 一把新的，舊的作廢。

準備一張「金鑰清單」

邊申請邊記，最後用生成器組成 .env

開一個純文字檔或記事本，申請到一把就貼一把，格式像這樣：

GEMINI_API_KEY = 你的Gemini金鑰
TAVILY_API_KEY = 你的Tavily金鑰
（GitHub / Render 用登入的，不是金鑰）

這張清單只留在你電腦。本章最後有「.env 生成器」，把這些貼進去就會幫你組好設定檔。

① Gemini · Google AI Studio

整個 App 的 AI 大腦

Google AI Studio 是 Google 提供的 Gemini 模型平台，可以免費申請 API 金鑰。我們的翻譯、拍照理解、語音朗讀、即時語音，全靠它。

🌐 網址：aistudio.google.com　（申請金鑰直達頁：aistudio.google.com/apikey）

需要一個 Google 帳號（Gmail 即可）。沒有的話先去 accounts.google.com 註冊一個。

進入網站、登入

  2. 右上角點「Sign in」，用你的 Google 帳號登入，同意服務條款。

Gemini 免費額度與花費上限

怎麼用才不會被扣款

Google AI Studio 有免費額度，課堂用量完全用不完。若你之後開放很多人使用，才可能需要付費方案。

  • 可在 AI Studio 設定「月支出上限（spending cap）」，避免暴衝的帳單。
  • 若某天呼叫回報 429 RESOURCE_EXHAUSTED，多半是額度或上限到了。
💡 新手建議：先用免費額度，需要更多再到 ai.studio/spend 調整上限。

Gemini 金鑰 · 驗收

確認你手上這把是對的
  • 金鑰是 AIza 開頭的一長串英數字。
  • 已完整複製、貼到你的金鑰清單。
  • 知道它會填到 GEMINI_API_KEY 和 GOOGLE_API_KEY。

④ Render · 讓網站上線

把你的 App 掛在雲端 24 小時待命

Render 是一個部署平台。你把程式碼放上 GitHub，Render 就能自動抓下來、跑起來，給你一個公開網址。

🌐 網址：render.com　·　有免費方案，足夠讓這個 App 上線。

最方便的註冊方式：直接用剛剛的 GitHub 帳號登入，之後選專案免設定。

✅ Render 不用申請 API 金鑰！只要用 GitHub 登入註冊一個 Render 帳號就好。（Render API key 是給「寫程式自動控制 Render」用的，本課用不到。）你要準備的金鑰是給 App 用的 GEMINI_API_KEY（＋選填 Tavily／OpenAI），部署時填進 Render 環境變數即可。

Render 步驟 1–2

用 GitHub 登入
  1. 打開 render.com，點「Get Started / Sign up」。
  2. 選「Sign up with GitHub」，授權 Render 連結你的 GitHub 帳號。

天氣、匯率：免金鑰

好消息，這兩個不用申請

我們刻意選了免金鑰的公開服務：

  • 天氣 → Open-Meteo（免費、免金鑰）。
  • 匯率 → ER-API / Frankfurter（免費、免金鑰）。

（選填）OpenAI 相容金鑰

想用 OpenAI／Groq／DeepSeek 才需要

翻譯預設用 Gemini（免費額度大）就很夠。若你想改用或加裝 OpenAI／Groq／DeepSeek 等相容服務，才需要它們的金鑰。

服務去哪申請備註
OpenAIplatform.openai.com → API keys → CreateAPI 要付費，記得設 Usage limit
Groqconsole.groq.com → API Keys免費、速度快
DeepSeekplatform.deepseek.com便宜
🔌 拿到後兩種填法：① 下載執行後在 App 右上 ⚙️ 設定 選「OpenAI 相容」填（每人各自、存自己瀏覽器）；② 想整站共用，部署時填進 Render 環境變數 OPENAI_API_KEY（CH18）。
⚠️ OpenAI 的 API 要付費（跟免費的 ChatGPT 網頁不同）。務必到帳戶設每月 Usage limit，避免帳單爆掉。只想免費就用 Gemini 或 Groq。

（選填）OpenWeatherMap

想要另一種天氣來源才需要

天氣預設用免金鑰的 Open-Meteo 就很好。若你想改用 OpenWeatherMap 當來源，可到 openweathermap.org 免費申請一把 API Key，填到設定即可。

⚠️ 新申請的 OpenWeatherMap 金鑰通常要等 10 分鐘～2 小時才生效。不急著用可先跳過。

🎁 .env 金鑰生成器

把你的金鑰填進去，一鍵組成設定檔
⚙️ 產生你的 .env
按「產生 .env」，這裡會出現你的 .env 內容…
存成專案根目錄的 .env 檔。此檔不要上傳 GitHub（CH18 會把它加進 .gitignore）。金鑰只留在你電腦。

.env 是什麼？怎麼用？

存金鑰的專用小檔案

.env 是一個純文字檔，一行一個「名稱=值」，專門放金鑰這類機密設定。程式啟動時會讀它，取得金鑰。

因為它含機密，所以絕不上傳 GitHub；雲端部署時改用平台的環境變數（CH18 教）。

✅ 剛剛生成器產生的內容，存成 .env 放專案根目錄，本機執行就會自動讀到。

金鑰之後放哪裡？

三個位置，各有用途
位置用途章節
.env 檔（本機）自己電腦上執行、測試本章生成器
App 設定面板使用者在網頁填自己的金鑰CH17
Render 環境變數上線後全裝置共用CH18

重點一致：金鑰永遠不寫死在會公開的程式碼裡。
